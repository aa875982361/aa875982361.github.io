其中我设置了 TAG funcName 主要是为了方便定位 错误信息

另外 还要注意日志的等级,写日志的时候,要分清楚是用来干嘛的,
例如

  1. 我要看某个变量的值,这是时候就用plog.d 只是调试才用到
  2. 如果 我要看返回回来的数据的话,可以用info 答应出来,因为这个可以看到 我获取到的信息有没有错误,问题是不是出在后台返回参数错误
  3. 如果 某个变量的值 类型为 undefined 不是正常业务逻辑,这个时候就要用 plog.w 打印出来,并且在正式环境不需要关掉
  4. 如果 发生系统级别的错误,那就要用 error 打印出来 我用得比较少

之所以分级别,更重要的原因是 方便后台日志数据库,将必要信息存储进数据库里面,可以快速定位错误
